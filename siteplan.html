Yosemite National Park Guide — Project Site Plan

A small, student-built guide with practical info for first-time visitors.

1) Naming Conventions

  • Project folder: project
  • This document: siteplan.html
  • CSS folder: styles → siteplan.css
  • Images folder: images → wireframe-mobile.png, wireframe-desktop.png
  • Naming style: lowercase, hyphen-separated file names (no spaces).

2) Site Name

Yosemite Pocket Guide

The name conveys a concise, practical guide focused on essential info for first‑time or returning visitors. It is easy to remember and fits the site’s scope.

3) Site Purpose

A small informational site with 3–4 pages—Home, Trails & Viewpoints, Plan Your Visit (passes, seasons, transport, lodging), and About/Contact—with shared navigation and a responsive, mobile‑first layout (GitHub Pages hosting).

Planned JavaScript Features

  • Trail filter/search by difficulty, distance, and area (data in arrays/objects, rendered dynamically).
  • Favorites / Trip list saved with localStorage.
  • Theme toggle (light/dark) remembered across visits.
  • Contact form validation with template‑string messages.
  • Lazy‑loaded, optimized images and accessible alt text.

Why this subject

I’ve visited Yosemite twice, and those trips showed me how much first‑time visitors need clear, practical information. Building a small guide lets me turn experience into something helpful while demonstrating course skills—semantic HTML, responsive CSS, and interactive JavaScript.

4) Scenarios (Target‑audience questions)

  • “Which beginner‑friendly trails have great views within a 2–3 hour window?”
  • “Do I need a peak‑season reservation or just the standard entrance pass?”
  • “Where can I park and use the shuttle if I arrive mid‑day?”
  • “What’s the best time of year to see waterfalls at their peak?”

5) Color Schema

The plan uses the selected colors below (also applied throughout this page):

  • #1F4D3A Forest Green — headings, buttons
  • #5B6B73 Granite Gray — body text accents
  • #6FA8DC Sky Blue — links, highlights

6) Typography

  • Headings: Poppins, 700/600 weight (applied on this document).
  • Body: Inter, 400/600 (applied on this document).
  • Fallback stack: system UI, Arial, sans-serif.

7) Wireframes (Home Page)

Sketches for the home page layout. One for mobile, one for a wider desktop view.

8) Testing & Validation Checklist

  • Passes W3C HTML validator (no errors).
  • Passes W3C CSS validator (no errors).
  • Good color contrast and accessible alt text.
  • Lighthouse: performance, best practices, and SEO reviewed.
